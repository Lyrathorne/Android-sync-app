# DeviceSync Keyboard privacy model

DeviceSync Keyboard является Android Input Method Editor и технически получает ввод только через предоставленный системой `InputConnection`.

## Что клавиатура делает

- вставляет нажатые символы в активное поле;
- читает не более 64 символов перед курсором для локальных подсказок;
- как выбранный default IME наблюдает текстовый Android clipboard;
- по opt-in настройке передаёт clipboard только через существующий аутентифицированный `ConnectionManager` доверенному Windows-устройству;
- хранит до 50 элементов clipboard локально в AES-GCM шифрованном виде с ключом Android Keystore;

## Что клавиатура не делает

- не открывает второй socket и не читает сетевой `InputStream`;
- не записывает нажатия, surrounding text, пароли или clipboard в логи;
- не отправляет набираемый текст для подсказок;
- не использует камеру;
- не использует микрофон и не запускает скрытую запись;
- не копирует код, словари, branding или ресурсы закрытых клавиатур.

## Sensitive mode

Для password, visible password, web password, number password, `IME_FLAG_NO_PERSONALIZED_LEARNING` и ручного incognito mode отключаются:

- подсказки и автокоррекция;
- пользовательское обучение;
- clipboard panel/history;
- чтение surrounding text сверх операций, необходимых Backspace/Enter.

При завершении ввода очищается ссылка на текущий sensitive context.

Ограничение Android: clipboard не содержит надёжной метки, из какого приложения или поля он был скопирован. DeviceSync не сохраняет clipboard, пока IME видит sensitive поле; дополнительно пользователь может полностью выключить историю или включить incognito mode. Приложения с паролями должны запрещать копирование секретов на своей стороне.

## Хранение и очистка

- Пользовательская история clipboard: SharedPreferences payload, зашифрованный AES/GCM; ключ неэкспортируемый в Android Keystore.
- Пользовательский словарь: локальные SharedPreferences, без контекста и частоты использования.
- Pending clipboard для reconnect: один последний текст в существующих приватных app preferences. Он очищается после успешной постановки сообщения на передачу.
- Историю и пользовательский словарь можно очистить из настроек клавиатуры.

DeviceSync не синхронизирует историю в облако. Удаление данных приложения удаляет локальную историю, словарь и настройки.
